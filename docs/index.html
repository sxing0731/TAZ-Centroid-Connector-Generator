Help

This tool is used to review and edit TAZ centroid connectors. A TAZ is a Traffic Analysis Zone. Each TAZ has a centroid, and the CC, or centroid connector, links that centroid to the roadway network through a valid network node. CCs should generally point toward areas with higher activity density inside or near the TAZ. In this run, HERE roadway density data was used as the main reference for identifying likely activity areas.

The number of CCs is somewhat subjective and depends on the size and shape of the TAZ. Smaller TAZs usually have 1 to 2 CCs, while larger TAZs may have 3. The automatic generator targets 1 to 3 CCs, at least 70 degrees apart, no more than 200 ft outside the TAZ, and no GSTDM crossing before the target node. During manual review, these automatic rules may be overridden, including the CC count. A target node can still be used by only one TAZ; if another TAZ already uses it, choose a nearby different node. Red major nodes remain blocked during preprocessing, but manual review may connect to one after acknowledging the Warning. CCs should avoid connecting to internal TAZ nodes: use a node within 200 ft of the TAZ boundary whenever one satisfies all hard rules. A farther internal node is labeled INTERIOR FALLBACK. CCs should also avoid connecting directly to freeways, major arterials, or intersections. The generation rules provide the default result, while the reviewer may make a documented manual exception, including major-node use. Cross-TAZ node sharing remains prohibited. MapLibre loads the OpenStreetMap basemap and local MVT vector tiles only for the visible map view.

Basic workflow

  1. Open the web tool.
  2. MapLibre renders TAZ boundaries, labels, nodes, and GSTDM links from local MVT vector tiles in WebGL. Low zooms use simplified links and clustered nodes; zoom 12 and above use the full network through overzoomed detail tiles.
  3. Select a TAZ from the list, double-click any TAZ polygon or number on the map, or use the TAZ search box to jump to a specific TAZ.
  4. Selecting a TAZ zooms to its boundary and highlights it; the other loaded layers remain available and reappear as you pan or zoom.
  5. Review the existing centroid connectors on the map.
  6. The active review dataset is GSTDM2025 Global TAZ / Global CC. Its browser-saved edits and uploaded CC baseline are separate from the previous New TAZ / New CC work. The 658 Global TAZs that also belong to the New TAZ dataset start as REVIEWED.
  7. To QC a coworker's Global CC export, click Load CC File. DBF and CSV can be selected directly; select SHP together with its same-name DBF. The uploaded CCs replace the Global CC baseline in this browser until you click Use Default CC.
  8. Click a connector to inspect its current connected node.
  9. To modify a connector, select the connector and click a valid target node on the map. The edit is saved immediately.
  10. To add a new connector, click Add CC, then choose the target node. The new connector is generated and saved directly.
  11. To add a missing roadway link, click Add Missing Links, choose the first node, then choose the second node. Any visible detailed node can be used, and candidate nodes highlight on hover.
  12. HERE_MISS is a separate map layer. Use Load HERE_Missing File to replace the current layer from a HERE_MISS DBF or CSV. Reverse-direction records are combined into one map link, and the operation can be undone.
  13. Use Export HERE_MISS to download DBF or CSV. New links default to two directional records with LANES=1, HERE_MISS=1, and FCLASS=32; these values can be edited in the Missing Links table.
  14. Use the CC, Missing Links, and TAZ Status tabs in the right panel to review records. Selecting a CC or HERE_MISS link on the map highlights its table row. Right-click a HERE_MISS line or table row to delete it. Drag the left edge of the right panel to resize the tables.
  15. Edit table values directly. Press Enter or leave a field to save it; press Escape to cancel. TAZ IDs and calculated CC counts remain read-only because they are tied to published geometry and live connector records.
  16. Use Undo Edit or Redo Edit if needed.
  17. Every TAZ with at least one CC starts as WAITING FOR QC. A TAZ with no CC is automatically FLAG. A TAZ changes to EDITED when uploaded CC data differs from the default CCs or when a user changes a connector, QC note, or other saved review data.
  18. Mark the TAZ as REVIEWED when the connectors look correct. Use Review + Next or Review + Previous to continue in either direction. Right-click a TAZ in the list or on the map to set WAITING FOR QC, FLAG, EDITED, or REVIEWED directly. A zero-CC TAZ always remains FLAG.
  19. Continue through the remaining TAZs.
  20. When review is complete, use Final CC Export, choose DBF or CSV, and choose whether to include the matching QCNOTES companion file with A, B, and QC_NOTES fields.
  21. Use Export TAZ QC Status to download all TAZ statuses and notes as DBF, CSV, or a polygon Shapefile ZIP. DBF is the default.
  22. To discard the Global review work and restore the published Global CC and HERE_MISS inputs, click Reset Browser Data and confirm. This does not delete browser-saved work for the previous New TAZ / New CC dataset.

Please focus on whether each centroid connector is connected to a reasonable valid node. Preprocessing does not connect to red major nodes with MAJOR_LEVEL <= 2. During manual editing, those red nodes may be selected, but a persistent warning must be reviewed and closed manually.

For the GitHub static web version, your edits are saved locally in your browser, so closing the webpage, closing the browser, or restarting your computer should not affect your saved work. As long as you reopen the same link in the same browser, your previous edits, QC notes, and reviewed status should still be available.
