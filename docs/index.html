Instructions for Users

This tool is used to review and edit TAZ centroid connectors. A TAZ is a Traffic Analysis Zone. Each TAZ has a centroid, and the CC, or centroid connector, links that centroid to the roadway network through a valid network node. CCs should generally point toward areas with higher activity density inside or near the TAZ. In this run, HERE roadway density data was used as the main reference for identifying likely activity areas.

The number of CCs is somewhat subjective and depends on the size and shape of the TAZ. Smaller TAZs usually have 1 to 2 CCs, while larger TAZs may have 3. Every TAZ must keep 1 to 3 CCs. CCs cannot extend more than 200 ft outside the TAZ or cross a GSTDM link before reaching the target node. A GSTDM target node can be used by only one TAZ; if another TAZ already uses it, choose a nearby different node. CCs within one TAZ must be at least 70 degrees apart; this angle is never reduced, so delete a lower-priority CC if needed. CCs should avoid connecting to internal TAZ nodes: use a node within 200 ft of the TAZ boundary whenever one satisfies all hard rules. A farther internal node is labeled INTERIOR FALLBACK. CCs should also avoid connecting directly to freeways, major arterials, or intersections. These rules have already been built into the program, so the main task is to human-review the generated results and make corrections where needed. The OpenStreetMap basemap loads only the tiles needed for the visible map view and can be turned on to help review whether the connector location makes sense.

Basic workflow

  1. Open the web tool.
  2. All TAZs, CCs, GSTDM nodes, and GSTDM links load once. The initial map zooms to the first TAZ; use Zoom All at any time to see the statewide view.
  3. Select a TAZ from the queue, click any TAZ polygon or number on the map, or use the TAZ search box to jump to a specific TAZ.
  4. Selecting a TAZ zooms to its boundary and highlights it; the other loaded layers remain available and reappear as you pan or zoom.
  5. Review the existing centroid connectors on the map.
  6. To QC a coworker's export, click Load CC File. DBF and CSV can be selected directly; select SHP together with its same-name DBF. The uploaded CCs replace the default CCs in this browser until you click Use Default CC.
  7. Click a connector to inspect its current connected node.
  8. To modify a connector, select the connector and click a valid target node on the map. The edit is saved immediately.
  9. To add a new connector, click Add CC, then choose the target node. The new connector is generated and saved directly.
  10. Use Undo Edit or Redo Edit if needed.
  11. Every TAZ starts as FLAG. A TAZ changes to EDITED when uploaded CC data differs from the default CCs or when a user changes a connector, QC note, or other saved review data.
  12. Mark the TAZ as REVIEWED when the connectors look correct. Right-click a TAZ in the queue or on the map to set FLAG, EDITED, or REVIEWED directly.
  13. Continue through the remaining TAZs.
  14. When review is complete, use Final CC Export, choose DBF or CSV, and choose whether to include the matching QCNOTES companion file with A, B, and QC_NOTES fields.
  15. Use Export TAZ QC Status to download all TAZ statuses and notes as CSV or as a polygon Shapefile ZIP.
  16. To discard all browser-saved edits and restore the original published dataset, click Reset Browser Data and confirm. This also removes uploaded CC data, QC notes, reviewed status, and custom layer order.

Please focus on whether each centroid connector is connected to a reasonable valid node. Major nodes with MAJOR_LEVEL <= 2 should not be used as snap nodes; only MAJOR_LEVEL 3, 4, or 5 are eligible.

For the GitHub static web version, your edits are saved locally in your browser, so closing the webpage, closing the browser, or restarting your computer should not affect your saved work. As long as you reopen the same link in the same browser, your previous edits, QC notes, and reviewed status should still be available.
